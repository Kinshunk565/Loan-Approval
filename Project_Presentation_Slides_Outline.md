### Slide 2: Problem Statement (The "Why")
*   Manual loan processing takes **hours or days**.
*   High risk of human error or individual bias.
*   The need for a data-driven, automated system to improve efficiency and reduce **NPAs** (Non-Performing Assets).

### Slide 3: Objectives
*   To build a predictive model to classify loan applications as "Approved" or "Rejected".
*   To identify key features (e.g., Credit History) affecting loan processing.
*   To compare different machine learning algorithms for accuracy.

### Slide 4: Proposed System / Technologies
*   **Supervised Machine Learning:** Binary Classification.
*   **Tech Stack:** Python, Pandas, NumPy, Scikit-learn, Matplotlib, Seaborn.
*   **Deployment:** Streamlit / Flask UI.

### Slide 5: Data Description (Input Features)
*   **Demographic:** Gender, Married, Dependents, Education.
*   **Financial:** Applicant Income, Co-applicant Income.
*   **Loan Details:** Loan Amount, Loan Term, Credit History, Property Area.

### Slide 6: Machine Learning Workflow
*   Data Collection (CSV).
*   Data Preprocessing (Handling missing values, Outlier removal).
*   Label Encoding & Feature Scaling.
*   Train-Test Split (80-20).

### Slide 8: Model Selection (Algorithms Used)
*   **Logistic Regression:** Base model for probability.
*   **Decision Trees:** Flowchart-based decision making.
*   **Random Forest:** Ensemble of trees for high accuracy (The chosen model).

### Slide 9: Evaluation Metrics
*   **Accuracy Score:** Overall % of correct predictions.
*   **Confusion Matrix:** Breakdown of TP, TN, FP, FN.
*   **Precision & Recall:** Balancing risky approvals vs. missed opportunities.
